# 自动化测试生成

粘贴代码或 API 文档，自动生成全面的测试用例，覆盖正常路径、边界和异常。

## 功能

- **单元测试**：自动覆盖所有函数，正常输入/边界/异常
- **API测试**：根据接口文档生成请求和响应测试
- **E2E测试**：描述用户操作流程 → 生成端到端测试脚本
- **Mock生成**：自动生成外部依赖的 Mock
- **覆盖率报告**：分析代码的测试覆盖盲区

## 使用方法

```
为这个函数生成全面的单元测试：[粘贴代码]

要求：用 Jest，覆盖正常/边界/异常情况
```

```
根据这个API文档生成接口测试：[粘贴 API 文档]

测试框架：pytest
需要验证：状态码、响应结构、鉴权
```

## 输出示例

```javascript
describe('calculateDiscount', () => {
  // 正常情况
  test('VIP用户打8折', () => { ... });
  
  // 边界情况
  test('价格为0时不报错', () => { ... });
  test('折扣为100%时返回0', () => { ... });
  
  // 异常情况
  test('负数价格抛异常', () => { ... });
  test('非数字价格抛异常', () => { ... });
});
```

## 支持框架

Jest / Vitest / pytest / JUnit / Go testing / RSpec / PHPUnit

## 技巧

- 告诉它代码的业务逻辑，生成的测试更贴合实际
- 让它"先生成测试用例清单，确认后再写代码"
- E2E测试描述操作步骤时像写用户故事一样
